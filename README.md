# Retail Hub Mobile

A full-stack mobile e-commerce application built with Flutter and Firebase.

> **Disclaimer:** All product images and names used in this project are for educational and demonstration purposes only. All rights belong to their respective owners. This project is a student portfolio piece and is not intended for commercial use.

---

## Tech Stack

**Frontend**

- Flutter (Dart)
- go_router (Declarative routing with redirect guards)
- Provider (State management)

**Backend**

- Firebase Authentication (Email/Password)
- Cloud Firestore (NoSQL database)

---

## Features

### Customer

- User registration and login with Firebase Authentication
- Browse and search/filter products by name, category, stock status, and price
- Product detail page with quantity selector and cached image loading
- Shopping cart with persistent state via shared preferences
- Checkout with order summary and stock deduction on order placement
- Order history

### Admin

- Protected admin dashboard (role-based access)
- Full product CRUD — create, edit, and delete products

## Getting Started

### Prerequisites

- Flutter SDK (Dart SDK `>=3.3.0 <4.0.0`)
- Android Studio with an Android emulator configured
- A Firebase project with **Authentication** (Email/Password) and **Cloud Firestore** enabled

### Installation

```
flutter pub get
```

### Firebase Setup

1. Create a project at [firebase.google.com](https://firebase.google.com)
2. Enable **Authentication** (Email/Password)
3. Enable **Cloud Firestore**
4. Run the FlutterFire CLI to generate `firebase_options.dart`:

```
flutterfire configure
```

> `firebase_options.dart` and `google-services.json` are excluded from this repo via `.gitignore`. You must generate your own.

### Running the App

Due to a Gradle build issue where `flutter run` fails to locate the generated `.apk` file, use the included PowerShell script instead.

In your terminal (PowerShell), from the project root:

```
.\run.ps1
```

This script manually builds the debug APK via Gradle, then launches it on the connected emulator using the built binary directly.

> **Note:** `flutter run` alone will produce the error: *"Gradle build failed to produce an .apk file."* Use `.\run.ps1` as the workaround.
